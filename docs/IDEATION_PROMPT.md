# 🧠 Hackathon Ideation Prompt — RPC Fast Infrastructure Credits

> Paste this into Gemini Deep Think, o3, or any deep reasoning model.

---

You are a hackathon strategist with a 90%+ win rate. Your job is to analyze a specific hackathon and generate 5 non-obvious, high-winability project ideas.

## Hackathon Context

- **Name**: Colosseum Frontier Hackathon — RPC Fast Infrastructure Credits Track
- **Platform**: Superteam Earn (Colosseum)
- **Theme / Focus**: Build a Solana project using high-performance RPC infrastructure. Winners get monthly RPC plan subscriptions (up to 3 months) — NOT cash.
- **Tracks & Sponsor Prizes**:
  - 1st Place (up to 5 teams): 2 months Aperture (~$5,000 value) or 3 months Focus
  - 2nd Place (up to 4 teams): 1 month Aperture (~$2,000 value) or 3 months Focus
  - 3rd Place (up to 4 teams): 1 month Stream (~$1,000 value) or 2 months Focus
  - Bonus (up to 4 teams): Must use RPC Fast during hackathon, 1 month Aperture
  - **Up to 17 total winners!**
- **Judging Criteria**: Technical Quality, Innovation, Impact, Demonstration
- **Duration**: ~3 weeks
- **Submission Requirements**: Colosseum submission, public GitHub, README, demo video/pitch deck, follow @rpcfast, join Telegram
- **Special Note**: NOT required to use RPC Fast infra (except Bonus Awards). Prize is infra credits not cash.

## RPC Fast Infrastructure (Free During Hackathon)
- 120M compute units/month
- 500 req/s RPC rate limit
- Unlimited bandwidth
- Shredstream / Yellowstone gRPC
- Frankfurt geo endpoint

## My Profile

- Solo developer
- Strong in: Next.js, React, Node.js, Supabase, Python, AI/ML APIs
- Can ship a polished MVP within 3 weeks
- Prefer projects that demo well on video

## Your Task

Generate **5 project ideas** ranked by winnability. For each idea:

1. **Name** — catchy, memorable
2. **One-Line Pitch** — what it does in one sentence
3. **Target Track** — RPC Fast Infrastructure Credits
4. **Why It Wins** — what makes this stand out (17 winner slots!)
5. **Secret Angle** — a non-obvious insight about RPC performance advantages
6. **Tech Stack** — specific tools/APIs to use (keep it lean)
7. **30-Second Demo Script** — what the judge sees in the first 30 seconds
8. **Risk** — what could go wrong and how to mitigate
9. **Difficulty** — Easy / Medium / Hard (for a solo dev)
10. **Docs Distance** — 🟢 Novel / 🟡 Adjacent / 🔴 Canonical. **Never recommend a 🔴 idea.**
11. **Winner Archetype** — Capability-unlock / Infrastructure / Visualization
12. **RPC Usage Strategy** — how to demonstrate RPC Fast usage for Bonus Award eligibility
13. **Production Plan** — how to deploy with real transactions
14. **Emotional Hook** — a single sentence framed as one specific person's problem
15. **Rubric Alignment Score** — score 0-10 against EACH: Technical Quality, Innovation, Impact, Demonstration. Reject if total < 70% of max.
16. **Scope Depth Rating** — Narrow+Deep / Wide+Shallow. **Prefer Narrow+Deep for solo.**
17. **Cross-Submission Potential** — which other Frontier tracks could this project ALSO win?

## Thinking Framework

Before generating ideas, reason through:

### Standard Analysis
- 17 winners from unknown submissions — VERY favorable odds
- Not required to use RPC Fast (except Bonus) = submit your main project here as a bonus
- Projects that DO use RPC Fast get an extra shot at the Bonus Award
- Shredstream/Yellowstone gRPC = real-time data streaming — build something latency-sensitive
- Focus on projects that would BENEFIT from 3 months of premium RPC post-hackathon

### Anti-Pattern Analysis
- **Cross-Submit Strategy**: This is a FREE BONUS track — submit your best project from another track
- **RPC Showcase**: If you use RPC Fast, highlight latency metrics in your demo
- **Infra Credits Value**: Only valuable if you plan to keep building post-hackathon
- **Frankfurt Geo**: Deploy your app in EU for lowest latency demo

### Emotional Hook Test (Mandatory)
For EACH idea, write one sentence starting with "A [specific person]..." that frames the problem as a human crisis.

### Seed Data Design Test
What is the ONE demo flow that shows RPC performance advantage?

## Output Format

Rank ideas from highest to lowest winnability. Be specific. Every idea must have a clear visual demo moment that makes a judge say "wow" in the first 10 seconds.

**Never recommend a 🔴 Docs Distance idea.**
**Never recommend an idea that fails the Emotional Hook Test.**
**For solo dev: prefer Narrow+Deep scope.**
**This is best used as a cross-submission track — submit your main project here too.**
